## Option A: npm (Recommended)

The easiest way to install flowState CLI is via npm. This works on Windows, macOS, and Linux.

<Steps>
  <Step title="Install via npm">
    ```bash
    npm install -g flowstate-cli
    ```
    
    The installer will automatically:
    - Detect your platform and architecture
    - Download the appropriate binary from GitHub Releases
    - Install it to your npm global bin directory
  </Step>
  
  <Step title="Run flowState">
    ```bash
    flowstate
    ```
    
    On first run, flowState will:
    1. Initialize SQLite database in `~/.flowstate/`
    2. Download the embedding model (~90MB)
    3. Launch the TUI interface
  </Step>
</Steps>

<Warning>
  **32-bit systems are not supported.** If you get an "ia32" or "x86" architecture error:
  
  1. You have 32-bit Node.js installed on a 64-bit system
  2. Uninstall Node.js and install the 64-bit version from [nodejs.org](https://nodejs.org/)
  3. Run `npm install -g flowstate-cli` again
</Warning>

### Linux/macOS: PATH Setup

If you get "flowstate: command not found" after installation:

<Tabs>
  <Tab title="Quick Fix">
    Use `npx` instead:
    ```bash
    npx flowstate
    ```
  </Tab>
  
  <Tab title="Permanent Fix">
    Add npm global bin directory to your PATH:
    
    **Bash** (`~/.bashrc`):
    ```bash
    export PATH="$(npm config get prefix)/bin:$PATH"
    ```
    
    **Zsh** (`~/.zshrc`):
    ```bash
    export PATH="$(npm config get prefix)/bin:$PATH"
    ```
    
    Then reload your shell:
    ```bash
    source ~/.bashrc  # or ~/.zshrc
    ```
  </Tab>
</Tabs>

## Option B: Download Binary

Download pre-built binaries from [GitHub Releases](https://github.com/Jericoz-JC/flowState-CLI/releases/latest).

### Supported Platforms

| Platform | Architecture | File |
|----------|--------------|------|
| Windows | x64 | `flowstate-windows-amd64.zip` |
| Windows | ARM64 | `flowstate-windows-arm64.zip` |
| macOS | Intel (x64) | `flowstate-darwin-amd64.tar.gz` |
| macOS | Apple Silicon (ARM64) | `flowstate-darwin-arm64.tar.gz` |
| Linux | x64 | `flowstate-linux-amd64.tar.gz` |
| Linux | ARM64 | `flowstate-linux-arm64.tar.gz` |

<Steps>
  <Step title="Download the binary">
    Go to [GitHub Releases](https://github.com/Jericoz-JC/flowState-CLI/releases/latest) and download the file for your platform.
  </Step>
  
  <Step title="Extract the archive">
    <CodeGroup>
      ```bash macOS/Linux
      tar -xzf flowstate-*.tar.gz
      ```
      
      ```powershell Windows
      Expand-Archive flowstate-*.zip
      ```
    </CodeGroup>
  </Step>
  
  <Step title="Run the binary">
    <CodeGroup>
      ```bash macOS/Linux
      ./flowstate
      ```
      
      ```powershell Windows
      .\flowstate.exe
      ```
    </CodeGroup>
  </Step>
  
  <Step title="(Optional) Add to PATH">
    Move the binary to a directory in your PATH for global access:
    
    <CodeGroup>
      ```bash macOS/Linux
      sudo mv flowstate /usr/local/bin/
      ```
      
      ```powershell Windows (Admin)
      Move-Item flowstate.exe C:\Windows\System32\
      ```
    </CodeGroup>
  </Step>
</Steps>

<Note>
  32-bit systems are not supported. All binaries are 64-bit only (x64 or ARM64).
</Note>

## Option C: Go Install

If you have Go 1.21+ installed, you can install flowState CLI directly:

```bash
go install github.com/Jericoz-JC/flowState-CLI/cmd/flowState@latest
```

This will:
- Download the source code
- Build the binary
- Install it to `$GOPATH/bin` (usually `~/go/bin`)

<Tip>
  Make sure `$GOPATH/bin` is in your PATH:
  ```bash
  export PATH="$GOPATH/bin:$PATH"
  ```
</Tip>

Then run:
```bash
flowState
```

## Option D: Build from Source

For developers who want full control or want to contribute:

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/Jericoz-JC/flowState-CLI
    cd flowState-CLI
    ```
  </Step>
  
  <Step title="Install dependencies">
    ```bash
    go mod download
    ```
  </Step>
  
  <Step title="Build the binary">
    ```bash
    go build -o flowstate ./cmd/flowState
    ```
    
    This creates a `flowstate` binary in the current directory.
  </Step>
  
  <Step title="Run flowState">
    ```bash
    ./flowstate
    ```
  </Step>
  
  <Step title="(Optional) Install globally">
    ```bash
    sudo mv flowstate /usr/local/bin/
    ```
  </Step>
</Steps>

## Troubleshooting

### 404 Errors During npm Install

<Steps>
  <Step title="Verify GitHub release exists">
    Check that the release exists at:
    ```
    https://github.com/Jericoz-JC/flowState-CLI/releases/tag/vX.Y.Z
    ```
  </Step>
  
  <Step title="Check binaries are uploaded">
    Ensure all 6 platform binaries are present in the release
  </Step>
  
  <Step title="Verify version match">
    npm package version in `package.json` must match the release tag exactly
  </Step>
</Steps>

### Old Version Runs Instead of New Version

<Tabs>
  <Tab title="npm">
    Uninstall and reinstall:
    ```bash
    npm uninstall -g flowstate-cli
    npm install -g flowstate-cli
    ```
  </Tab>
  
  <Tab title="Binary">
    Check which binary is being used:
    ```bash
    # macOS/Linux
    which flowstate
    
    # Windows
    where flowstate
    ```
    
    Remove old binaries from your PATH.
  </Tab>
</Tabs>

### Binary Not Found After npm Install

The npm `postinstall` script may have failed. Try:

1. Check that binaries are downloadable from GitHub Releases
2. Reinstall:
   ```bash
   npm uninstall -g flowstate-cli
   npm cache clean --force
   npm install -g flowstate-cli
   ```
